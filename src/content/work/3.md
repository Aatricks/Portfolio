LightDiffusion-Next is a refactored, performance-oriented generation system that combines a Python generation core, a FastAPI server, and a React frontend. 

### Architecture & Improvements
The refactor prioritized architectural improvements, maintainability, and backend/server integration.
- `src/user/pipeline.py` routes generation settings through a shared Context/Pipeline flow instead of keeping logic trapped inside one UI.
- `server.py` handles request batching, queueing, seed control, upload handling, model and ControlNet discovery, and WebSocket preview streaming.
- `ModelFactory.py` detects SD1.5, SDXL, and Flux-style layouts and can assemble separate diffusion, text-encoder, and VAE paths when needed.

This repo focuses on local-performance work and practical workflow tooling, built to be used locally and integrated deeply, rather than only benchmarked.
