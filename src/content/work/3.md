LightDiffusion-Next combines a Python generation core, a FastAPI server, a React frontend, and still-supported Streamlit and Gradio interfaces into one local-generation stack.

- `src/user/pipeline.py` routes generation settings through a shared Context/Pipeline flow instead of keeping logic trapped inside one UI
- `server.py` adds request batching, queueing, seed control, upload handling, model and ControlNet discovery, and WebSocket preview streaming
- `ModelFactory.py` detects SD1.5, SDXL, and Flux-style layouts and can assemble separate diffusion, text-encoder, and VAE paths when needed
- the repo is built around local-performance work rather than a single demo path: Stable-Fast, multiscale diffusion, DeepCache, token merging, CFG variants, prompt enhancement, and Dockerized GPU deployment

The repo is strongest where performance work meets product surface. It is built to be used locally, not only benchmarked.
