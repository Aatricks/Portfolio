`llmedge` is an Android-native inference orchestration and runtime platform. It bridges the gap between raw C++ ML runtimes and usable Android product APIs.

### Architecture & Constraints
Managing memory constraints, model acquisition, validation, and lifecycle-aware coordination on diverse mobile hardware is technically hard. 
- `LLMEdge` exposes text, speech, image, vision, and RAG clients behind a Kotlin-first API surface, managing native pools and backend selection (CPU/GPU/NPU fallbacks) under the hood.
- `ModelRepository` keeps artifact selection, validation, and caching separate from inference.
- The runtime stack uses `RuntimePool` and `RuntimeCoordinator` to acquire engines and retry across OpenCL, Vulkan, and CPU when a backend fails.
- Balancing binary size vs. backend coverage and maintaining stable APIs across changing underlying engines are core design tradeoffs.

### Ownership & Outcome
I owned the end-to-end architecture, native bindings, Kotlin wrapping, and documentation. It is a mature engineering platform with JVM tests, Android instrumentation, and opt-in native E2E coverage, tested on real devices.
