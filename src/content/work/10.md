EasyReader is built like a full Android product. Compose navigation, Room persistence, Hilt DI, source adapters, local-file ingestion, and on-device summarization all feed the same reading flow.

- `MainActivity` routes between reader, library, and explore screens while also handling deep links, shared URLs, and local document picks
- `ContentRepository` normalizes four content paths: remote web chapters, PDFs, EPUBs, and local HTML files, with cache inspection and prefetch support
- `ExploreRepository` fans out across source adapters such as NovelFire and MangaBat, while `LibraryRepository` persists progress, reading modes, chapter grouping, update checks, and cleanup logic in Room
- `SummaryService` integrates `llmedge` to download a local Qwen3 GGUF model and generate chapter summaries on device instead of sending text to a remote service

This repo matters because it combines product polish with real Android state management, offline behavior, and source-integration work.
