This repo is more careful than most “battery optimizer” scripts. The interesting part is not a huge list of tweaks; it is the safety model around them.

- `AdbClient` and `SubprocessRunner` use argv-based subprocess calls instead of shell-built command strings
- `StateRecorder` snapshots settings, `device_config`, app ops, standby buckets, and enabled states before mutation
- the default path is intentionally narrow and documented, while riskier flows are separated into explicit experimental menus
- whitelist management, saved rollback state, and dry-run support make it practical to test changes without guessing how to undo them later

The repo is best described as a reversible ADB operations tool for battery-related experiments, not a magic battery-life script.
